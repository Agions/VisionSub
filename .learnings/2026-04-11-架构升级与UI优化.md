## 一、frontend-design-pro CSS 规范修复

### 修复的问题

| 文件 | 问题描述 | 修复方案 |
|------|----------|----------|
| `Button.vue` | `rgba(var(--error), 0.12)` CSS 变量不能传入 rgba() | `color-mix(in oklch, var(--error) 12%, transparent)` |
| `Button.vue` | SCSS `lighten($primary, 8%)` 无法处理 CSS 变量 | `color-mix(in oklch, var(--primary) 85%, white)` |
| `StatusBar.vue` | `statusPulse` 动画过于复杂（scale + box-shadow） | 简化为纯 opacity 动画 |
| `SidePanel.vue` | `action-btn--primary` 渐变使用 SCSS lighten | 改用 color-mix + CSS 变量 |
| `_mixins.scss` | `btn-danger` mixin 同样 rgba(var()) 问题 | 统一 color-mix() |
| `global.scss` | link hover `lighten()` | 改用 color-mix() |

### 核心教训

**CSS 变量不能直接用于 SCSS 颜色函数：**
- `rgba(var(--color), 0.5)` — ❌ CSS 变量是字符串，不是颜色值
- `lighten($color, 10%)` — ❌ SCSS 函数无法处理 CSS 变量
- **正确做法：** 使用 `color-mix(in oklch, var(--color) 90%, white)`

## 二、核心架构模块化升级

### 背景

原架构问题：
- `useOCREngine.ts` 职责过重（700+行）
- 后处理算法散落在 Extractor 和 OCREngine 两处
- 场景检测耦合在提取器中，无法独立测试
- 导出逻辑放在 Store 里（违反单一职责）
- 双重置信度校准函数（`calibrate` + `calibrateEnhanced`）

### 新架构 (`src/core/`)

```
src/core/
├── index.ts                  ← Barrel export
├── SubtitlePipeline.ts       ← 4阶段后处理管道
├── SceneDetector.ts          ← 量化直方图场景检测
├── SubtitleExporter.ts       ← 9种格式导出引擎
└── ConfidenceCalibrator.ts   ← 统一置信度校准
```

### SubtitlePipeline — 4阶段后处理

```
Stage 1: filterJitter    ← 移除单帧 OCR 噪声
Stage 2: mergeSplit      ← 合并因场景跳跃而分裂的相同字幕
Stage 3: mergeSimilar    ← 合并时间接近的相似字幕
Stage 4: computeEndTime  ← 根据下一条字幕计算 endTime
```

关键设计：
- Levenshtein 相似度计算带 **LRU 缓存**（最多 2000 条）
- 每阶段为纯函数，输入输出均为 `SubtitleLite[]`
- 可单独禁用或调整参数

### SceneDetector — 量化直方图

算法：RGB 通道各量化到 16 bins → chi-square 比较

```typescript
// 采样 500 像素，构建 48 bins 直方图
// chi-square > threshold → 场景变化
```

优势：
- O(n) 时间复杂度（n = 采样像素数）
- 对 JPEG 压缩噪声鲁棒（量化平滑）
- 无状态设计，支持切换算法

### SubtitleExporter — 9种格式

| 格式 | 路径 |
|------|------|
| SRT, VTT, ASS, SSA | `SubtitleExporter.ts` |
| JSON, TXT, LRC, SBV, CSV | `SubtitleExporter.ts` |
| ~~STL, TTML~~ | 已移除（未使用） |

### ConfidenceCalibrator — 校准信号

```typescript
interface CalibrationSignal {
  type: 'penalty' | 'bonus'
  factor: number   // 乘数，如 0.80
  reason: string
}
```

语言特定规则：
- **CJK**: 孤立单字、引号不平衡
- **Latin**: 全大写、孤立数字、句子结尾标点
